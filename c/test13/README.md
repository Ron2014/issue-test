# test13: 静态链接库的循环引用/依赖 

## 问题来源

这个探讨是在为石器时代服务器（逻辑服gmsv）搭建开发环境时出现的。

石器时代服务器是纯C项目，编译使用的纯makefile，分三个步骤

1. make clean
2. make depend
3. make

编译过程大体可以解释成

1. *.c 文件生成 *.o 文件
2. 同目录下的 *.o 文件打包成 *.a 静态链接库
3. 最后一步将子目录生成的所有 *.a 和根目录下的 *.o 链接生成二进制可执行程序 gmsvjt

这个过程，步骤1、2可以正常通过，执行到步骤3时出现刷屏的`undefined reference` 错误，为此，我将make报错grep `: undefined reference to ` ，然后再对文件做排序、去重

```bash
$ grep ': undefined reference to ' log > log1
$ cat log1 | awk -F':' '{printf "%s:%s\n", $1, $3}' | sort | uniq > log2
$ sed -Ei 's/\ undefined\ reference\ to\ //' log2
```

PS. 输出格式都是类似这样

```bash
/mnt/e/Gitee/stone-age/server/engine/code/gmsv/char/char_base.c:3746: undefined reference to `_ITEM_CHECKINDEX'
```

```c
INLINE double time_diff(struct timeval subtrahend,struct timeval subtractor);
```

`INLINE` 的定义放在 common.h 中

```c
#define INLINE inline
```

所以我们的测试用例，也要考虑到到这是个**内联函数**。
